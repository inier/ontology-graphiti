# ADR-037: 前端移动优先、响应式设计与国际化策略

> **来源**: 前端架构设计

**状态**: 已接受 | **日期**: 2026-04-17

---

## 1. 上下文

随着移动设备的普及和全球化业务的需求，前端平台需要支持：
- 多种设备类型：移动端、平板、笔记本、桌面 PC
- 多种屏幕尺寸和分辨率
- 多语言环境（中文、英文等）
- 不同的交互方式（触摸、鼠标、键盘）
- 充分组件化，项目做好组件分级管理

---

## 2. 决策

### 2.1 核心原则：移动优先（Mobile First）

采用 **移动优先（Mobile First）** 的响应式设计策略：

1. **从小到大设计**：先为移动设备设计核心布局和交互，再为更大屏幕增强体验
2. **渐进增强**：移动端提供基础功能，平板和桌面端逐步增强
3. **内容优先**：移动端优先展示关键内容，复杂功能在大屏幕展示

### 2.2 响应式断点设计

| 设备类型 | 断点范围 | 布局策略 | 侧边栏 | 内容列数 |
|----------|----------|----------|--------|----------|
| **手机** | < 576px | 单栏全屏 | 底部 Tab 导航 | 1 列 |
| **平板（竖屏）** | 576-768px | 单栏+浮动面板 | 可收起侧边栏 | 1-2 列 |
| **平板（横屏）** | 768-1024px | 双栏布局 | 可折叠侧边栏 | 2 列 |
| **笔记本** | 1024-1440px | 标准两栏布局 | 固定侧边栏 | 2-3 列 |
| **桌面 PC** | 1440-1920px | 宽松两栏布局 | 固定侧边栏 | 3 列 |
| **超大屏** | ≥ 1920px | 三栏布局 | 固定侧边栏 | 3+ 列 |

### 2.3 网格系统

采用 **12 列网格系统**：

| 设备 | 列数 | 列宽 | 间距 | 容器最大宽度 |
|------|------|------|------|--------------|
| 手机 | 4 | auto | 12px | 100% |
| 平板 | 8 | auto | 16px | 720px |
| 桌面 | 12 | auto | 24px | 1200px |
| 大屏 | 12 | auto | 24px | 1440px |

### 2.4 组件响应式策略

| 组件类型 | 移动端 | 平板 | 桌面 |
|----------|--------|------|------|
| 导航 | 底部 Tab | 侧边栏收起 | 侧边栏展开 |
| 表格 | 卡片式列表 | 简化表格 | 完整表格 |
| 图谱 | 全屏+缩放 | 分屏+侧边详情 | 分屏+侧边详情+工具栏 |
| 地图 | 全屏+浮动按钮 | 分屏+侧边面板 | 分屏+侧边面板+工具栏 |
| 表单 | 单列堆叠 | 双列 | 多列+高级选项 |

### 2.5 国际化（i18n）策略

#### 支持语言

| 语言 | Locale Code | 状态 |
|------|-------------|------|
| 简体中文 | zh-CN | 当前默认 |
| 英语 | en-US | 规划中 |
| 日语 | ja-JP | 规划中 |

#### 国际化实现

```
frontend/
├── src/
│   ├── locales/           # 语言包目录
│   │   ├── zh-CN/        # 简体中文
│   │   │   └── messages.json
│   │   └── en-US/        # 英语
│   │       └── messages.json
│   ├── i18n/             # i18n 配置
│   │   └── index.ts
│   └── components/        # 组件使用 t('key') 或 useTranslation()
```

#### 数字/日期格式化

| 类型 | 中文 | 英文 |
|------|------|------|
| 数字 | 1,234.56 | 1,234.56 |
| 货币 | ¥1,234.56 | $1,234.56 |
| 日期 | 2026年4月17日 | April 17, 2026 |
| 时间 | 14:30:45 | 2:30:45 PM |

## 3. 技术实现

### 3.1 CSS 策略

采用 **CSS Grid + Flexbox** 结合 **CSS 变量** 实现响应式：

```css
:root {
  /* 颜色 */
  --color-primary: #1890ff;
  --color-bg: #f0f2f5;

  /* 间距 */
  --spacing-xs: 4px;
  --spacing-sm: 8px;
  --spacing-md: 16px;
  --spacing-lg: 24px;

  /* 字体 */
  --font-size-sm: 12px;
  --font-size-md: 14px;
  --font-size-lg: 16px;
}

@media (min-width: 768px) {
  :root {
    --font-size-sm: 12px;
    --font-size-md: 14px;
    --font-size-lg: 16px;
  }
}
```

### 3.2 媒体查询断点

```css
/* 手机 */
@media (max-width: 575px) { }

/* 平板竖屏 */
@media (min-width: 576px) and (max-width: 767px) { }

/* 平板横屏 */
@media (min-width: 768px) and (max-width: 1023px) { }

/* 笔记本 */
@media (min-width: 1024px) and (max-width: 1439px) { }

/* 桌面 */
@media (min-width: 1440px) and (max-width: 1919px) { }

/* 大屏 */
@media (min-width: 1920px) { }
```

### 3.3 React 响应式 Hook

```typescript
import { useMediaQuery } from 'antd';

const useBreakpoint = () => {
  const isMobile = useMediaQuery('(max-width: 575px)');
  const isTablet = useMediaQuery('(min-width: 576px) and (max-width: 1023px)');
  const isDesktop = useMediaQuery('(min-width: 1024px)');

  return { isMobile, isTablet, isDesktop };
};
```

## 4. 性能优化

### 4.1 移动端优化

| 优化项 | 策略 |
|--------|------|
| 首屏加载 | 代码分割、懒加载 |
| 图片 | WebP 格式、响应式图片 srcset |
| 图谱 | 低性能设备简化渲染 |
| 交互 | 防抖、节流 |

### 4.2 按需加载

```
路由懒加载:
/ontology  →  本体图谱模块（包含 G6）
/map       →  态势地图模块（包含 Leaflet）
/timeline  →  时间线模块
```
