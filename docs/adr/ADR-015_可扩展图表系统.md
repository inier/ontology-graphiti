# ADR-015: 可扩展图表系统

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

**状态**: 已接受 | **日期**: 2026-04-16

---

## 1. 上下文

系统需要展示多种类型的图表：
- 关系图谱
- 统计图表
- 时间线
- 地理空间图

需要统一的图表框架。

---

## 2. 决策

采用 **ECharts + AntV G6** 作为图表库。

### 2.1 图表类型映射

| 图表类型 | 库 | 用途 |
|---------|-----|------|
| 折线/柱状 | ECharts | 统计图表 |
| 关系图谱 | AntV G6 | 节点关系 |
| 时间线 | AntV G6 | 事件序列 |
| 地理空间 | CesiumJS | 地理位置 |

### 2.2 统一接口

```python
class ChartRenderer(ABC):
    @abstractmethod
    def render(self, container: str, data: dict, options: dict): ...

class ChartManager:
    def register(chart_type: str, renderer: ChartRenderer): ...
    def render(chart_type: str, ...): ...
```

---

## 3. 后果

| 正面 | 负面 |
|------|------|
| ✅ 丰富的图表类型 | ❌ 多库学习成本 |
| ✅ 统一接口 | ❌ 包体积增加 |
| ✅ 易于扩展 | |

---

## 4. 可逆性

**中**。可统一替换为单一图表库。
