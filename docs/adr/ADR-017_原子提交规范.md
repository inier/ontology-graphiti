# ADR-017: 原子提交规范

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

**状态**: 已接受 | **日期**: 2026-04-16

---

## 1. 上下文

需要规范代码提交行为，确保：
- 提交原子性（一个提交一个功能）
- 提交信息清晰
- 易于回滚

---

## 2. 决策

制定**原子提交规范**。

### 2.1 提交格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 2.2 Type 类型

| Type | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug 修复 |
| docs | 文档更新 |
| style | 代码格式 |
| refactor | 重构 |
| test | 测试 |
| chore | 构建/工具 |

---

## 3. 后果

| 正面 | 负面 |
|------|------|
| ✅ 提交历史清晰 | ❌ 需要团队遵守规范 |
| ✅ 易于追踪问题 | ❌ 自动化工具支持 |
| ✅ 便于生成 CHANGELOG | |

---

## 4. 可逆性

**高**。规范可调整。
