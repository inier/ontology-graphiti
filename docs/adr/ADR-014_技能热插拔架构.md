# ADR-014: 技能热插拔架构

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

**状态**: 已接受 | **日期**: 2026-04-16

---

## 1. 上下文

系统需要支持动态加载技能（Skills），无需重启服务：
- 运行时加载新技能
- 卸载不需要的技能
- 技能版本管理

---

## 2. 决策

构建**技能热插拔架构**，基于插件机制。

### 2.1 架构

```
┌─────────────────────────────────────────────────────────────┐
│                   SkillRegistry                                 │
├─────────────────────────────────────────────────────────────┤
│  SkillLoader │ SkillWatcher │ SkillValidator                 │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 核心接口

```python
class Skill(ABC):
    name: str
    version: str

    @abstractmethod
    def execute(self, context: dict) -> dict: ...

class SkillRegistry:
    def register(skill: Skill) -> bool: ...
    def unregister(skill_name: str) -> bool: ...
    def get(skill_name: str) -> Skill: ...
    def list_all() -> List[Skill]: ...
```

---

## 3. 后果

| 正面 | 负面 |
|------|------|
| ✅ 技能动态加载 | ❌ 需要插件签名验证 |
| ✅ 无需重启服务 | ❌ 版本兼容性管理复杂 |
| ✅ 易于扩展 | |

---

## 4. 可逆性

**高**。可降级为静态技能加载。
