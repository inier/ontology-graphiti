# ADR-026: 采用MCP协议作为外部系统集成标准

## 状态

**已接受** | **创建日期**: 2026-04-13 | **最后更新**: 2026-04-13

## 上下文

Graphiti系统需要与多种外部战场仿真系统集成，包括：
1. **战场仿真器**（Unreal Engine等）
2. **雷达模拟器**（信号生成、探测模拟）
3. **气象数据源**（实时天气、预报数据）
4. **卫星影像服务**
5. **友军位置更新系统**

这些系统具有不同的接口协议、数据格式和通信模式，导致以下问题：
- **接口碎片化**: 每个系统都有自定义的API，难以统一管理
- **集成复杂度高**: 需要为每个系统开发专用的适配器代码
- **维护成本高**: 系统升级时需要同步更新多个适配器
- **扩展性差**: 新增系统需要重新设计集成架构

现有的解决方案包括：
1. **自定义REST API**: 每个系统定义自己的API标准
2. **消息队列集成**: 使用Kafka/RabbitMQ进行异步通信
3. **RPC框架**: 使用gRPC/Thrift等RPC框架
4. **API网关**: 通过API网关统一路由

但这些方案都存在一定的局限性，特别是在AI Agent工具调用场景下。

## 决策

**采用Model Context Protocol (MCP) v1.0作为Graphiti系统与外部战场仿真系统的标准集成协议。**

### 核心决策点：
1. **协议标准化**: 所有外部系统都通过MCP协议接入
2. **统一工具接口**: 将外部能力封装为统一的AI Agent工具
3. **安全隔离**: MCP Server在独立进程中运行，提供安全边界
4. **动态扩展**: 支持运行时动态添加/移除MCP Server
5. **开发友好**: 提供丰富的开发工具和调试支持

## 替代方案分析

### 方案A: 自定义REST API（当前状态）
- **优点**: 简单直接，易于理解和调试
- **缺点**: 
  - 接口碎片化严重
  - 缺乏统一的安全模型
  - 难以支持实时数据流
  - 工具发现和元数据管理复杂

### 方案B: gRPC + Protobuf
- **优点**: 
  - 高性能，支持流式传输
  - 强类型接口定义
  - 多语言支持良好
- **缺点**:
  - 缺乏标准的工具发现机制
  - 安全模型需要自行设计
  - 与AI Agent工具注册表集成复杂

### 方案C: 消息队列（Kafka/RabbitMQ）
- **优点**:
  - 支持异步和批量处理
  - 解耦系统间依赖
  - 高吞吐量
- **缺点**:
  - 不适合请求-响应模式
  - 实时性较差
  - 工具调用语义不直观

### 方案D: MCP协议（最终选择）
- **优点**:
  - **专门为AI Agent设计**: 原生支持工具和资源发现
  - **标准化接口**: 统一的工具调用和资源访问接口
  - **安全隔离**: 内置沙箱执行和权限控制
  - **实时支持**: 原生支持WebSocket流式传输
  - **生态完善**: 有丰富的工具链和社区支持
- **缺点**:
  - 相对较新的协议，生态系统仍在发展中
  - 需要外部系统实现MCP Server接口

## 影响

### 正面影响
1. **降低集成复杂度**: 统一协议简化了系统集成
2. **提升安全性**: 沙箱隔离和标准化安全模型
3. **增强可扩展性**: 支持动态添加新系统
4. **改善开发体验**: 统一的开发工具和调试支持
5. **提高可观测性**: 统一的监控和日志体系

### 负面影响
1. **迁移成本**: 现有系统需要实现MCP Server接口
2. **学习曲线**: 团队需要学习MCP协议和工具链
3. **运行时开销**: 额外的协议转换层可能增加延迟

## 详细设计

### 架构设计
```
┌─────────────────────────────────────────────────────────┐
│                  Graphiti 核心系统                         │
│  ┌─────────────────┐  ┌─────────────────┐              │
│  │  OpenHarness    │  │  Tool Registry  │              │
│  │     Agent       │  │    (43+工具)    │              │
│  └────────┬────────┘  └────────┬────────┘              │
│           │                    │                        │
│           └────────────────────┼────────────────────┐   │
│                                ▼                    │   │
│                    ┌─────────────────────┐          │   │
│                    │  MCP Protocol Layer │          │   │
│                    │  • Server Manager   │          │   │
│                    │  • Protocol Adapter │          │   │
│                    │  • Message Router   │          │   │
│                    └─────────────────────┘          │   │
│                                │                    │   │
│         ┌──────────────────────┼────────────────────┘   │
│         ▼                      ▼                        │
│  ┌──────────────┐    ┌─────────────────┐                │
│  │ 战场仿真器    │    │ 雷达模拟器      │                │
│  │ MCP Server   │    │ MCP Server     │                │
│  └──────────────┘    └─────────────────┘                │
│         │                      │                        │
│         ▼                      ▼                        │
│  ┌──────────────┐    ┌─────────────────┐                │
│  │  Unreal      │    │ 雷达信号生成     │                │
│  │  Engine      │    │ 模拟器          │                │
│  └──────────────┘    └─────────────────┘                │
└─────────────────────────────────────────────────────────┘
```

### 核心组件
1. **MCP Server Manager**: 管理所有注册的MCP Server实例
2. **Protocol Adapter**: 实现MCP v1.0协议栈
3. **Message Router**: 消息分发和负载均衡
4. **Security Layer**: 认证、授权、加密、审计
5. **Monitoring & Metrics**: 性能监控和告警

### 数据流
1. **工具调用**: Agent → Tool Registry → MCP Protocol Layer → MCP Server → 外部系统
2. **数据订阅**: 外部系统 → MCP Server → MCP Protocol Layer → WebSocket → Agent
3. **资源访问**: Agent → MCP Protocol Layer → MCP Server → 外部资源

## 实施计划

### 阶段1: 协议框架实现（1-2周）
- 实现MCP v1.0协议栈核心功能
- 开发MCP Server管理器和协议适配器
- 建立基本的认证和安全机制

### 阶段2: 战场仿真器集成（2-3周）
- 实现战场仿真器MCP Server
- 集成Unreal Engine仿真能力
- 测试工具调用和实时数据流

### 阶段3: 雷达模拟器集成（2-3周）
- 实现雷达模拟器MCP Server
- 集成雷达信号生成和探测模拟
- 测试性能和可靠性

### 阶段4: 其他系统集成（2-3周）
- 实现气象数据源MCP Server
- 集成卫星影像和友军位置服务
- 完善监控和运维工具

### 阶段5: 优化和部署（1-2周）
- 性能优化和压力测试
- 安全审计和渗透测试
- 生产环境部署和监控

## 成功指标

### 技术指标
1. **协议兼容性**: 100%通过MCP v1.0规范测试
2. **性能指标**: 
   - 平均延迟 < 100ms
   - 吞吐量 > 1000请求/秒
   - 连接稳定性 > 99.9%
3. **安全指标**: 
   - 安全事件数 = 0
   - 认证成功率 > 99.9%
   - 审计日志覆盖率 = 100%

### 业务指标
1. **集成效率**: 新系统集成时间减少50%
2. **维护成本**: 系统维护工作量减少30%
3. **开发效率**: 新工具开发时间减少40%
4. **系统稳定性**: 集成相关故障减少70%

## 备注

本决策将作为Graphiti系统外部集成的基础标准，所有新的外部系统集成都必须遵循MCP协议标准。现有系统的迁移将按优先级分阶段进行。

**免责声明**: MCP协议仍在快速发展中，本决策基于MCP v1.0规范。如果协议有重大变更，将重新评估本决策。
