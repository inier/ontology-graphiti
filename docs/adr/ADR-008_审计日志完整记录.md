# ADR-008: 审计日志完整记录

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

**状态**: 已接受 | **日期**: 2026-04-16

---

## 1. 上下文

系统需要完整记录审计日志，用于：
- 操作追溯
- 合规审计
- 问题排查

---

## 2. 决策

构建**完整审计日志系统**。

### 2.1 审计内容

| 操作类型 | 记录内容 |
|---------|---------|
| 权限变更 | 用户、角色、时间 |
| 数据访问 | 用户、实体、时间 |
| 配置变更 | 用户、配置项、旧值、新值 |
| 系统事件 | 时间、类型、详情 |

### 2.2 存储

- 结构化存储（Elasticsearch）
- 日志保留期：1年

---

## 3. 后果

| 正面 | 负面 |
|------|------|
| ✅ 操作可追溯 | ❌ 存储成本 |
| ✅ 合规审计 | ❌ 性能影响 |
| ✅ 问题排查 | |

---

## 4. 可逆性

**高**。可调整日志级别。
