# ADR-038: 本体管理引擎与用户认知引擎架构

> **状态**: 已接受
> **日期**: 2026-04-18
> **作者**: 软件架构师
> **相关**: ADR-007, ADR-037, ARCHITECTURE.md

---

## 1. 上下文

现有系统存在以下问题：

- **管理端**：从信息采集（数据摄入）到本体图谱构建的过程是黑盒，缺乏可视化审计界面
- **用户端**：尚未提供用户认知引擎，无法站在不同使用角色的角度展示可解释的处理过程

参考悦点科技 Knora 平台的"Ontology+大模型+规则引擎"复合架构，需要构建两个核心引擎来解决这些问题。

---

## 2. 决策

### 2.1 本体管理引擎

**核心功能**：将信息采集到本体图谱构建的黑盒过程透明化，提供可视化审计界面。

**架构组件**：

1. **Ingestion Audit**：数据摄入审计
   - 记录数据来源、处理过程、转换规则
   - 支持数据质量追踪和异常检测

2. **Ontology Builder**：本体构建器
   - 实体提取、关系识别、属性映射
   - 可视化展示构建过程

3. **Version Control**：版本控制
   - 版本追踪、变更对比、回滚机制
   - 支持本体版本管理

4. **Validation Engine**：验证引擎
   - 数据质量检查、一致性验证、完整性验证
   - 确保本体质量

5. **Audit Dashboard**：审计仪表盘
   - 实时监控、历史审计、异常告警
   - 提供管理端可视化界面

**技术实现**：
- 后端：Python FastAPI 服务
- 前端：React + Ant Design 仪表盘
- 存储：Neo4j + Graphiti

### 2.2 用户认知引擎

**核心功能**：构建用户端界面，站在不同使用角色的角度，观测到自己的问题被意图识别处理到最后返回结果的可解释过程。

**架构组件**：

1. **Intent Recognizer**：意图识别
   - 意图识别、需求解析、角色识别
   - 理解用户问题的真实意图

2. **Knowledge Navigator**：知识导航
   - 知识导航、推理路径、本体查询
   - 基于本体进行知识检索和推理

3. **Explanation Engine**：解释引擎
   - 过程解释、可解释性、可视化解释
   - 生成可理解的处理过程解释

4. **Role-based View**：基于角色的视图
   - 指挥官视图、情报分析员视图、操作员视图
   - 为不同角色提供定制化界面

5. **Result Presenter**：结果呈现
   - 结果呈现、多模态展示、交互反馈
   - 以用户友好的方式展示结果

**技术实现**：
- 后端：Python 服务 + LLM 集成
- 前端：React + Ant Design 界面
- 数据：Graphiti 知识图谱

---

## 3. 设计原则

### 3.1 可观测性
- **全流程可追溯**：每个数据处理步骤都有详细的审计记录
- **实时监控**：提供实时的本体构建状态监控
- **异常告警**：及时发现和处理数据质量问题

### 3.2 可解释性
- **透明推理**：展示 AI 如何基于本体进行推理
- **过程可视化**：以可视化方式展示处理过程
- **多维度解释**：从不同角度解释决策依据

### 3.3 可扩展性
- **模块化设计**：各组件独立可扩展
- **插件机制**：支持自定义插件和扩展
- **API 标准化**：提供标准化的 API 接口

### 3.4 安全性
- **访问控制**：基于角色的访问控制
- **数据保护**：敏感数据加密存储
- **审计日志**：完整的操作审计

---

## 4. 架构集成

### 4.1 与现有架构的集成

| 现有组件 | 集成点 | 说明 |
|----------|--------|------|
| Graphiti | 存储层 | 作为本体存储和查询引擎 |
| OPA | 权限控制 | 提供策略治理和权限管理 |
| OpenHarness | 协同编排 | 集成到 Agent 系统 |
| Skills | 工具调用 | 作为领域工具使用 |
| 前端界面 | 用户交互 | 提供管理端和用户端界面 |

### 4.2 数据流

```
外部信息 → 数据采集层 → 本体管理引擎 → Graphiti 存储 → 用户认知引擎 → 用户界面
```

### 4.3 服务依赖

| 服务 | 依赖关系 | 说明 |
|------|----------|------|
| 本体管理引擎 | Graphiti, Neo4j | 本体存储和查询 |
| 用户认知引擎 | 本体管理引擎, LLM | 知识检索和推理 |
| 前端界面 | 本体管理引擎, 用户认知引擎 | 提供用户交互 |

---

## 5. 实现路径

### 5.1 阶段 1：基础设施
- 设计本体管理引擎数据模型
- 开发审计接口和事件收集系统
- 构建用户认知引擎基础框架

### 5.2 阶段 2：核心功能
- 实现数据摄入审计功能
- 开发本体构建可视化界面
- 构建用户认知引擎核心组件
- 集成到现有架构

### 5.3 阶段 3：界面开发
- 开发管理端审计仪表盘
- 构建用户端交互界面
- 实现角色基于的视图

### 5.4 阶段 4：测试与优化
- 端到端测试
- 性能优化
- 用户体验改进

---

## 6. 影响评估

### 6.1 正面影响
- **透明度提升**：本体构建过程透明化
- **可解释性增强**：用户可以理解 AI 决策过程
- **管理效率**：提供管理端可视化工具
- **用户体验**：为不同角色提供定制化界面

### 6.2 风险评估
- **复杂度增加**：系统架构更加复杂
- **性能影响**：增加了额外的处理层
- **集成风险**：需要与现有系统深度集成

### 6.3 缓解策略
- **模块化设计**：降低系统复杂度
- **性能优化**：采用缓存和异步处理
- **渐进式集成**：分阶段集成到现有系统

---

## 7. 结论

通过构建本体管理引擎和用户认知引擎，系统将实现：

1. **管理端**：从信息采集到本体图谱构建的全流程可视化审计
2. **用户端**：基于角色的可解释 AI 处理过程展示
3. **架构优化**：提升系统的可观测性和可解释性
4. **用户体验**：为不同角色提供定制化的交互界面

这符合 Knora 平台的"Ontology+大模型+规则引擎"复合架构理念，将显著提升系统的可用性和用户满意度。
