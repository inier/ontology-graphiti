# ADR-029: 采用统一工具注册表架构管理AI Agent工具

## 状态

**已接受** | **创建日期**: 2026-04-13 | **最后更新**: 2026-04-13

## 上下文

Graphiti作为领域仿真AI Agent平台，需要支持多种类型的工具：
1. **文件系统工具**: 文件读写、目录操作、文件搜索
2. **网络工具**: HTTP请求、WebSocket通信、网络诊断
3. **数据处理工具**: 数据转换、分析、可视化
4. **AI/ML工具**: 模型推理、文本处理、图像识别
5. **系统工具**: 进程管理、系统监控、资源管理
6. **仿真专用工具**: 领域仿真、雷达模拟、气象分析

当前工具管理面临以下挑战：
- **工具分散**: 工具实现分散在各个模块，缺乏统一管理
- **接口不一致**: 不同工具的参数格式、返回值格式不一致
- **依赖管理复杂**: 工具之间的依赖关系难以管理
- **版本控制缺失**: 工具版本更新和兼容性管理困难
- **安全风险**: 工具执行缺乏统一的安全控制和审计
- **性能监控不足**: 工具使用频率、性能指标缺乏监控

现有管理方式包括：
1. **硬编码工具**: 在代码中直接实现工具函数
2. **配置文件管理**: 通过配置文件定义工具
3. **插件系统**: 使用插件架构管理工具
4. **服务发现**: 通过服务发现机制管理工具

## 决策

**采用统一工具注册表架构管理所有AI Agent工具，实现工具的统一注册、发现、执行和监控。**

### 核心决策点：
1. **统一工具接口**: 所有工具遵循相同的接口规范
2. **集中注册管理**: 工具在启动时注册到统一注册表
3. **动态发现加载**: 支持运行时动态发现和加载工具
4. **版本控制支持**: 支持工具多版本并存和版本切换
5. **安全执行环境**: 提供工具执行的安全沙箱和权限控制
6. **完整监控体系**: 实现工具使用统计、性能监控和错误追踪

## 替代方案分析

### 方案A: 硬编码工具（当前状态）
- **优点**: 
  - 简单直接，易于实现
  - 编译时检查，类型安全
  - 性能开销小
- **缺点**:
  - 工具分散，难以统一管理
  - 接口不一致，使用复杂
  - 扩展性差，新增工具需要代码修改
  - 缺乏版本控制和依赖管理

### 方案B: 配置文件管理
- **优点**:
  - 配置与代码分离
  - 支持动态配置更新
  - 相对简单易用
- **缺点**:
  - 配置文件可能变得复杂
  - 缺乏工具发现机制
  - 版本控制和依赖管理困难
  - 安全控制能力有限

### 方案C: 插件系统
- **优点**:
  - 支持动态加载和卸载
  - 插件隔离，降低耦合
  - 扩展性强
- **缺点**:
  - 实现复杂，需要完整的插件框架
  - 插件间依赖管理复杂
  - 版本兼容性问题
  - 安全风险较高

### 方案D: 服务发现（如Consul/Etcd）
- **优点**:
  - 支持服务动态发现
  - 高可用和负载均衡
  - 适合分布式部署
- **缺点**:
  - 不适合函数级工具管理
  - 工具调用语义不直观
  - 缺乏工具元数据管理
  - 安全控制复杂

### 方案E: 统一工具注册表（最终选择）
- **优点**:
  - **统一管理**: 所有工具集中管理
  - **标准接口**: 统一的工具调用接口
  - **动态发现**: 支持运行时工具发现
  - **版本控制**: 完善的多版本管理
  - **安全执行**: 内置安全沙箱和权限控制
  - **完整监控**: 工具使用统计和性能监控
- **缺点**:
  - 实现复杂度较高
  - 需要额外的运行时开销
  - 需要配套的开发工具链

## 影响

### 正面影响
1. **提高开发效率**: 统一的工具开发接口和工具链
2. **增强系统可扩展性**: 支持动态添加新工具
3. **改善工具质量**: 统一的测试、监控和优化
4. **提升系统安全性**: 统一的权限控制和审计
5. **加强运维能力**: 完整的工具监控和告警

### 负面影响
1. **系统复杂性增加**: 需要完整的工具生命周期管理
2. **性能开销**: 额外的工具调用开销
3. **学习成本**: 开发人员需要学习新的工具开发模式
4. **迁移成本**: 现有工具需要迁移到新架构

### 核心组件
1. **Tool Metadata Manager**: 管理工具元数据和版本信息
2. **Tool Loader**: 动态加载和实例化工具
3. **Tool Executor**: 执行工具调用，处理参数和返回值
4. **Tool Monitor**: 监控工具运行状态和性能
5. **Tool Repository**: 存储工具定义和实现

### 工具定义规范
```python
@dataclass
class ToolDefinition:
    """工具定义"""
    name: str                    # 工具名称（唯一标识）
    version: str                 # 工具版本（语义化版本）
    description: str             # 工具描述
    category: ToolCategory       # 工具分类
    security_level: SecurityLevel  # 安全等级
    parameters: List[ToolParameter]  # 参数定义
    return_type: Type            # 返回值类型
    dependencies: List[str]      # 依赖工具列表
    tags: List[str]              # 标签列表
    author: str                  # 作者信息
    license: str                 # 许可证
    created_at: datetime         # 创建时间
    updated_at: datetime         # 更新时间
```

### 工具分类体系
```python
class ToolCategory(Enum):
    """工具分类"""
    FILE_SYSTEM = "file_system"      # 文件系统工具
    NETWORK = "network"              # 网络工具
    DATA_PROCESSING = "data_processing"  # 数据处理工具
    AI_ML = "ai_ml"                  # AI/机器学习工具
    SYSTEM = "system"                # 系统工具
    SECURITY = "security"            # 安全工具
    MONITORING = "monitoring"        # 监控工具
    UTILITY = "utility"              # 实用工具
    SIMULATION = "simulation"        # 仿真工具
```

### 工具安全等级
```python
class SecurityLevel(Enum):
    """安全等级"""
    UNTRUSTED = "untrusted"      # 不可信工具（完全沙箱）
    TRUSTED = "trusted"          # 可信工具（受限访问）
    PRIVILEGED = "privileged"    # 特权工具（系统访问）
    ADMIN = "admin"              # 管理员工具（完全访问）
```

### 工具注册流程
1. **工具定义**: 开发者创建工具定义文件（YAML/JSON）
2. **代码实现**: 实现工具的核心逻辑
3. **测试验证**: 编写单元测试和集成测试
4. **构建打包**: 构建工具包，包含代码和资源
5. **签名验证**: 对工具包进行数字签名
6. **注册发布**: 注册到工具注册表，发布版本
7. **监控运维**: 监控工具使用情况，持续优化

## 示例用例

### 工具定义示例
```yaml
# domain_simulate_attack.tool.yaml
name: domain.simulate_attack
version: 1.0.0
description: 模拟领域攻击行动
category: simulation
security_level: trusted
author: "Graphiti Team"
license: "Apache-2.0"
tags: ["domain", "simulation", "combat"]
dependencies: ["domain.get_scenario@^1.0.0"]

parameters:
  - name: attacker_id
    type: string
    description: 攻击方单位ID
    required: true
    
  - name: target_id
    type: string
    description: 目标单位ID
    required: true
    
  - name: weapon_type
    type: string
    description: 武器类型
    required: true
    enum: ["missile", "artillery", "air_strike"]
    
  - name: strategy
    type: string
    description: 攻击策略
    default: "direct"
    enum: ["direct", "flank", "ambush"]

returns:
  type: object
  properties:
    success: boolean
    damage_percentage: number
    casualties: number
```

### 工具使用示例
```python
# 注册工具
registry = ToolRegistry()
registry.register_tool("domain.simulate_attack", "1.0.0", tool_impl)

# 执行工具
executor = ToolExecutor(registry)
result = await executor.execute(
    tool_name="domain.simulate_attack",
    parameters={
        "attacker_id": "unit-001",
        "target_id": "unit-002",
        "weapon_type": "missile",
        "strategy": "flank"
    },
    context={
        "user_id": "user-001",
        "session_id": "session-001"
    }
)

# 监控工具使用
monitor = ToolMonitor(registry)
stats = monitor.get_tool_stats("domain.simulate_attack")
print(f"工具使用次数: {stats.total_requests}")
print(f"平均执行时间: {stats.avg_execution_time_ms}ms")
```
