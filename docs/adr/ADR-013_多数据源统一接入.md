# ADR-013: 多数据源统一接入

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

**状态**: 已接受 | **日期**: 2026-04-16

---

## 1. 上下文

系统需要对接多种数据源：
- 传感器数据
- 外部 API
- 数据库
- 文件系统

需要统一的接入方式。

---

## 2. 决策

构建**统一数据接入层**，提供标准接口。

### 2.1 架构

```
┌─────────────────────────────────────────────────────────────┐
│                   DataSourceAdapter                             │
├─────────────────────────────────────────────────────────────┤
│  SensorAdapter │ APIAdapter │ DBAdapter │ FileAdapter     │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
              ┌───────────────────────┐
              │  统一数据格式         │
              │  (DataChunk)         │
              └───────────────────────┘
```

### 2.2 接口定义

```python
class DataSourceAdapter(ABC):
    @abstractmethod
    def connect(self) -> bool: ...

    @abstractmethod
    def fetch(self, query: dict) -> List[DataChunk]: ...

    @abstractmethod
    def disconnect(self): ...
```

---

## 3. 后果

| 正面 | 负面 |
|------|------|
| ✅ 统一的数据接入方式 | ❌ 适配器开发工作量 |
| ✅ 易于扩展新数据源 | ❌ 需要统一的错误处理 |
| ✅ 数据格式标准化 | |

---

## 4. 可逆性

**中**。可降级为单一数据源。
