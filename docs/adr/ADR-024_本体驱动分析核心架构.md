# ADR-024: 本体驱动分析核心架构

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

**状态**: 已接受 | **日期**: 2026-04-16

---

## 1. 上下文

系统需要以本体为核心驱动，实现：
- 实体标准化
- 关系建模
- 推理增强

---

## 2. 决策

构建**本体驱动分析架构**。

### 2.1 架构

```
┌─────────────────────────────────────────────────────────────┐
│                   OntologyDrivenAnalyzer                         │
├─────────────────────────────────────────────────────────────┤
│  本体层 │ 推理层 │ 分析层                                  │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 核心能力

| 能力 | 说明 |
|------|------|
| 实体标准化 | 基于本体定义验证和规范化实体 |
| 关系建模 | 自动建立实体间关系 |
| 推理增强 | 基于本体规则推理新知识 |

---

## 3. 后果

| 正面 | 负面 |
|------|------|
| ✅ 分析结果一致 | ❌ 本体维护成本 |
| ✅ 推理能力 | ❌ 复杂性增加 |
| ✅ 可解释性 | |

---

## 4. 可逆性

**中**。可降级为规则引擎。
