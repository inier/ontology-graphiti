# ADR-023: 多工作空间隔离架构

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

**状态**: 已接受 | **日期**: 2026-04-16

---

## 1. 上下文

系统需要支持多工作空间，实现：
- 数据隔离
- 权限隔离
- 独立配置

---

## 2. 决策

构建**多工作空间隔离架构**。

### 2.1 架构

```
┌─────────────────────────────────────────────────────────────┐
│                   Workspace A        │  Workspace B        │
│  - 独立数据                        │  - 独立数据          │
│  - 独立权限                        │  - 独立权限          │
│  - 独立配置                        │  - 独立配置          │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 隔离层级

| 层级 | 隔离内容 |
|------|---------|
| 数据 | Neo4j 数据库分区 |
| 权限 | OPA 策略隔离 |
| 配置 | 独立配置文件 |
| 日志 | 独立日志文件 |

---

## 3. 后果

| 正面 | 负面 |
|------|------|
| ✅ 数据完全隔离 | ❌ 跨工作空间查询复杂 |
| ✅ 权限独立管理 | ❌ 资源消耗增加 |
| ✅ 易于迁移 | |

---

## 4. 可逆性

**中**。可合并工作空间。
