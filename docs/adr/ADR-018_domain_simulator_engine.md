# ADR-018: 领域模拟推演引擎（Domain Simulator Engine）

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

---

**状态**: 已接受

**上下文**: 管理人员需要模拟推演功能，对选定方案进行沙盘验证，支持手工控制事件时间线，辅助决策。

**决策**: 设计领域模拟推演引擎（SimulatorEngine），基于事件驱动架构，支持：
- 预定义事件模板（遭遇、变化、响应等）
- 随机事件生成（基于概率分布）
- 事件时间线控制（加速/减速/暂停）
- 事件审核队列

**与 Mock 数据生成的区别**:

| 概念 | 职责 | 模块位置 |
|------|------|---------|
| **Mock 数据生成** | 为构建领域本体生成样本数据（静态） | `odap/biz/ontology/mock_data/` |
| **领域模拟推演** | 对选定方案进行沙盘验证（动态），辅助决策 | `odap/biz/simulator/` |

**后果**:
- ✅ 支持无外部数据源的演示和培训
- ✅ 可控的测试场景生成
- ✅ 支持边界条件和异常场景测试
- ✅ 支持方案验证（What-if Analysis）
- ❌ 需要维护事件模板库
- ❌ 模拟数据与真实数据的边界需明确区分

**可逆性**: 高。模拟器可独立部署或禁用。

## Phase 3 扩展（ADR-031/032，2026-04-14）

本 ADR 的设计在 Phase 3 中通过以下两个新 ADR 大幅扩展：

| 扩展 ADR | 内容 |
|---------|------|
| **ADR-031** | 为模拟器新增 Web 可视化服务（时间线/关系图谱/态势地图）+ 实时本体热写入管道（无需重启服务） |
| **ADR-032** | 定义统一的 OntologyDocument JSON 标准格式（基础属性/统计属性/规则/约束/关系/行动六层结构），所有数据统一沉淀 |

**新增的能力**（在原有随机生成基础上）：
- 联网检索真实资料 → LLM 归纳 → 标准本体格式
- Web 界面手动输入动态信息或按涉事方自动生成
- 界面导入/导出 `.odoc.json` 数据包
- 写入数据自动触发 Graphiti 图谱扩展（热生效，无需重启）
