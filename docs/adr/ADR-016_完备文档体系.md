# ADR-016: 完备文档体系

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

**状态**: 已接受 | **日期**: 2026-04-16

---

## 1. 上下文

系统需要完整的文档体系，包括：
- 架构文档
- 设计文档
- API 文档
- 使用指南

确保知识传承和团队协作。

---

## 2. 决策

构建**四层文档体系**。

### 2.1 文档层次

| 层次 | 文档 | 受众 |
|------|------|------|
| 架构层 | ARCHITECTURE.md, RESTRUCTURE_PLAN.md | 架构师 |
| 设计层 | docs/modules/*/DESIGN.md | 开发者 |
| 参考层 | docs/adr/*.md (ADR) | 决策者 |
| 使用层 | README.md, API docs | 用户 |

### 2.2 文档规范

- Markdown 格式
- 中英双语术语
- 版本化管理
- 自动生成 API 文档

---

## 3. 后果

| 正面 | 负面 |
|------|------|
| ✅ 知识传承 | ❌ 维护成本 |
| ✅ 团队协作 | ❌ 需要文档审查 |
| ✅ 新人 onboarding | |

---

## 4. 可逆性

**高**。文档可随时补充。
