# ADR-002: Graphiti 作为双时态知识图谱

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

**状态**: 已接受 | **日期**: 2026-04-16

---

## 1. 上下文

需要支撑"当时发生了什么"的历史回溯和时间区间查询，要求：
- 记录每个实体状态的时间维度
- 支持 What-if 分析（回溯历史状态）
- 支持时序推理和 RAG 增强

---

## 2. 决策

**Graphiti** 作为 Memory 层，**Neo4j** 作为底层图数据库。

### 2.1 架构

```
┌─────────────────────────────────────────────────────────────┐
│                      Graphiti Layer                              │
│  - 双时态知识图谱 (valid_time + transaction_time)            │
│  - Episode 管理                                              │
│  - RAG 增强检索                                              │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                      Neo4j Layer                                │
│  - 底层图存储                                                │
│  - Cypher 查询                                               │
│  - 索引优化                                                  │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 核心特性

| 特性 | 说明 |
|------|------|
| 双时态 | 原生支持 valid_time（业务时间）和 transaction_time（系统时间） |
| Episode | 事件驱动的知识组织，与领域事件流天然匹配 |
| RAG | 基于时序的语义检索，增强 LLM 上下文 |

---

## 3. 后果

| 正面 | 负面 |
|------|------|
| ✅ 原生支持双时态查询 | ❌ Graphiti 相对新兴，社区较小 |
| ✅ Episode 设计匹配领域事件流 | ❌ 需适配 OpenHarness Memory 接口 |
| ✅ 支持时序推理和 RAG 增强 | |

---

## 4. 可逆性

**中**。替换为纯 Neo4j 或其他图数据库需重写记忆层。
