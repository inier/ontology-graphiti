# ADR-010: 多模态文档处理可配置

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

**状态**: 已接受 | **日期**: 2026-04-16

---

## 1. 上下文

需要支持上传外部情报文档（图像、PDF、语音），面临：
- 多格式解析需求
- 不同敏感度数据需要选择本地/云端模型
- 多模型集成的灵活性

---

## 2. 决策

多模态处理采用**可插拔架构**，支持配置不同模型。

### 2.1 架构

```
┌─────────────────────────────────────────────────────────────┐
│                   DocumentPipeline                               │
├─────────────────────────────────────────────────────────────┤
│  文档解析层 │ 内容理解层 │ 本体更新层                      │
│  (可配置)   │ (可配置)   │ (固定)                         │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 可配置项

| 层级 | 可选模型 |
|------|---------|
| 图像识别 | Claude / GPT-4V / 本地 LLaVA |
| PDF 解析 | Adobe / 开源 PDF.js |
| 语音识别 | Whisper / 讯飞 / 阿里云 |
| 文本摘要 | GPT-4 / Claude / GLM |

---

## 3. 后果

| 正面 | 负面 |
|------|------|
| ✅ 灵活适配不同场景 | ❌ 配置复杂度增加 |
| ✅ 可根据数据敏感度选择本地/云端模型 | ❌ 多模型集成测试工作量大 |

---

## 4. 可逆性

**中**。模型可替换，但接口需要统一。
