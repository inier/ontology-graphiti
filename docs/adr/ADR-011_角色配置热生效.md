# ADR-011: 角色配置热生效

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

**状态**: 已接受 | **日期**: 2026-04-16

---

## 1. 上下文

系统中存在多种角色（指挥官、分析员、观察员等），需要：
- 动态调整角色权限
- 无需重启服务
- 权限变更即时生效

---

## 2. 决策

采用**配置热加载机制**，基于 OPA 策略引擎实现。

### 2.1 架构

```
┌─────────────────────────────────────────────────────────────┐
│                   角色配置管理层                                  │
├─────────────────────────────────────────────────────────────┤
│  配置文件 (JSON/YAML) → OPA Rego → 权限决策               │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 热加载流程

1. 修改配置文件
2. 触发 reload 事件
3. OPA 重新加载策略
4. 新权限即时生效

---

## 3. 后果

| 正面 | 负面 |
|------|------|
| ✅ 权限变更即时生效 | ❌ 需要 OPA 支持 |
| ✅ 无需重启服务 | ❌ 配置复杂度增加 |
| ✅ 策略可版本化管理 | |

---

## 4. 可逆性

**高**。可降级为静态权限配置。
