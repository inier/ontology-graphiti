# ADR-019: 多模态文档处理流水线

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

**状态**: 已接受 | **日期**: 2026-04-16

---

## 1. 上下文

需要上传外部情报文档（PDF、Word、图片等），并基于此更新本体。

---

## 2. 决策

构建**多模态文档处理流水线**（DocumentPipeline）。

### 2.1 流水线架构

```
┌─────────────────────────────────────────────────────────────┐
│                   DocumentPipeline                               │
├─────────────────────────────────────────────────────────────┤
│  文档解析层 │ 内容理解层 │ 本体更新层                      │
│  - PDF 解析  │ - LLM 摘要 │ - Graphiti 写入              │
│  - OCR      │ - 实体识别 │ - Neo4j 更新                  │
│  - 文本提取  │ - 关系抽取 │                               │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 处理流程

1. **文档解析**: PDF → 文本 / 图片 → OCR
2. **内容理解**: LLM 摘要、实体识别、关系抽取
3. **本体更新**: Graphiti 节点/边创建

---

## 3. 后果

| 正面 | 负面 |
|------|------|
| ✅ 支持多种文档格式 | ❌ 处理延迟较高 |
| ✅ 可独立配置各层模型 | ❌ 需要处理解析错误 |
| ✅ 流水线可监控和回滚 | |

---

## 4. 可逆性

**中**。可简化处理流程或降级为纯文本处理。
