# ADR-007: 前端采用 React + Ant Design 技术栈

> **来源**: `docs/ARCHITECTURE.md` 第 17 章

**状态**: 已接受 | **日期**: 2026-04-16

---

## 1. 上下文

需要构建领域前端和管理后台，面临技术选型决策：
- 需要企业级 UI 组件
- 需要图谱可视化能力
- 需要地理空间展示
- 需要快速迭代

---

## 2. 决策

采用 **React 19 + TypeScript + Ant Design 6** 作为前端技术栈。

### 2.1 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| React | 19 | UI 框架 |
| TypeScript | 5.x | 类型安全 |
| Ant Design | 6.x | 组件库 |
| AntV G6 | latest | 图谱可视化 |
| ECharts | latest | 图表 |
| CesiumJS | latest | 地理空间 |

### 2.2 架构

```
┌─────────────────────────────────────────────────────────────┐
│                        前端应用                                     │
├─────────────────────────────────────────────────────────────┤
│  React 19 + TypeScript + Ant Design 6                      │
├─────────────────────────────────────────────────────────────┤
│  AntV G6 │ ECharts │ CesiumJS                             │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                      后端 API (odap/web/)                        │
└─────────────────────────────────────────────────────────────┘
```

---

## 3. 后果

| 正面 | 负面 |
|------|------|
| ✅ 企业级组件库，开发效率高 | ❌ 包体积较大 |
| ✅ TypeScript 类型安全 | ❌ 学习曲线（对新成员） |
| ✅ AntV G6 + ECharts 支持图谱可视化 | |
| ✅ CesiumJS 支持地理空间展示 | |

---

## 4. 可逆性

**中**。前端框架切换成本较高，但组件层可抽象。
