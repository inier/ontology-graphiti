## 2. 整体架构概述

### 2.1 架构核心理念（参考 Knora 平台）

Knora 平台的核心理念是 **"知识驱动"** 和 **"可解释AI"**，我们将其适配为 ODAP 架构：

1. **知识结构化优先**：以本体为核心，先定义再应用
2. **全流程可解释**：从数据到决策，每个步骤都可追溯
3. **多角色可扩展**：不同业务场景快速适配
4. **基础设施可复用**：Graphiti/OPA/Skill 作为核心引擎

## 3. 核心分层设计

### 3.1 分层原则

1. **单一职责**：每层只负责一类问题
2. **标准接口**：层间通过标准化API交互
3. **可替换**：每层内部实现可独立优化
4. **可扩展**：每层支持水平扩展和垂直扩展

### 3.2 各层职责详析

| 层级 | 核心职责 | 关键模块 | 对应需求编号 |
|------|----------|----------|--------------|
| **用户交互层** | 提供各角色界面，处理用户输入与展示 | 指挥官UI、管理员UI、Chat UI | FR-1103, FR-1104 |
| **用户认知引擎层** | 意图识别、知识导航、解释引擎、角色视图 | 意图识别器、知识导航器、解释引擎 | FR-1100~FR-1104 |
| **Agent编排层** | 三Agent协同、任务调度、消息路由 | Commander Agent、Intelligence Agent、Operations Agent、编排器 | FR-001, FR-400~FR-401 |
| **核心引擎层** | Graphiti知识图谱、Skill注册、OPA策略 | Graphiti、Skill Registry、OPA Engine | FR-002~FR-004, FR-300~FR-301, FR-700 |
| **本体管理引擎层** | 数据审计、构建可视化、版本控制、验证 | 数据摄入审计、本体构建器、版本管理器、验证引擎 | FR-1000~FR-1003 |
| **工作空间隔离层** | 场景隔离、数据隔离、配置隔离 | Workspace Manager、隔离机制 | FR-100~FR-102, NFR-S03 |
| **基础设施与存储层** | 持久化存储、缓存、文件存储 | Neo4j、PostgreSQL、Redis、MinIO | 基础支撑 |

## 5. 技术实现文档

### 5.1 UI 设计规范

#### 5.1.1 设计原则

| 原则 | 描述 | 实现方式 |
|------|------|----------|
| 移动优先 | 先设计移动端，再适配桌面端 | ADR-037 + 响应式工具 |
| 角色定制 | 不同角色界面有所区别 | MOD-02 角色视图管理器 |
| 可视化主导 | 过程可视化、结果可视化 | 图表系统 + 图谱可视化 |
| 交互友好 | 快速反馈、操作可撤销 | 状态管理 + 回滚机制 |

#### 5.1.2 组件层级（参考 COMPONENT_HIERARCHY.md）

```
L5: 应用级 (Dashboard/Workspaces)

L3: 区域级 (Sidebar/ContentFooter)
L2: 功能级 (GraphView/Chart)

```
