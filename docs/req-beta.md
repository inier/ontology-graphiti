## 1. 项目概述

### 1.1 战略定位（核心调整）

> **⚠️ 重要声明**: 本系统是一个**通用的本体驱动分析决策平台**，战争分析只是当前建设的**一个实例化场景**。系统设计充分考虑多场景支持，通过工作空间机制实现场景隔离，任何领域都可以基于本平台快速构建专属的分析决策应用。

```
┌─────────────────────────────────────────────────────────────────────┐
│                     本体驱动分析决策平台 (ODAP)                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────┐    ┌─────────────┐    ┌─────────────┐           │
│   │  战争分析   │    │  金融分析   │    │  医疗诊断   │    ...    │
│   │   场景      │    │   场景      │    │   场景      │           │
│   └──────┬──────┘    └──────┬──────┘    └──────┬──────┘           │
│          │                  │                  │                    │
│   ┌──────┴──────────────────┴──────────────────┴──────┐              │
│   │                  本体层 (Ontology)               │              │
│   │   ┌─────────┐ ┌─────────┐ ┌─────────┐           │              │
```

### 1.2 核心价值

| 价值维度 | 描述 |
|----------|------|
| **通用性** | 不绑定任何垂直领域，支持任意领域的本体建模 |
| **可扩展性** | Skill、工作空间、场景完全可插拔 |
| **隔离性** | 多工作空间/多场景数据完全隔离 |
| **知识驱动** | 以本体为核心，实现语义级别的理解与推理 |
| **可视化** | 本体图谱可视化、过程解释、问答图表 |

### 1.3 典型应用场景

| 场景 | 领域本体 | 核心问题 |
|------|----------|----------|
| **战争分析（当前实例）** | 部队、装备、地形、气象 | 打击决策、态势评估 |
| **金融风控** | 账户、交易、资产、市场 | 风险评估、反欺诈 |
| **医疗诊断** | 症状、疾病、药物、患者 | 辅助诊断、治疗方案 |
| **供应链优化** | 供应商、物流、库存、订单 | 优化调度、风险预警 |
| **舆情分析** | 事件、媒体、观点、实体 | 趋势预测、影响评估 |

### 19. 模拟推演能力详细需求

#### 19.1 核心功能需求

**F-604: 沙箱隔离推演**
- **需求描述**: 提供完全隔离的沙箱环境，支持在Web界面配置参数，模拟不同方案，预演结果
- **技术实现**: Docker容器隔离，临时数据存储，生产环境零影响
- **优先级**: P0
- **验收标准**: 
  - 推演不影响生产数据
  - 支持多方案并行推演
  - 推演环境可清理回收

**F-605: 版本化管理**
- **需求描述**: 每个方案支持独立的临时存储和版本管理，支持回退
- **技术实现**: Git-style版本控制，参数快照，差异对比
- **优先级**: P0
- **验收标准**:
  - 支持任意版本回退
  - 可视化版本差异
  - 版本历史可追溯

**F-606: Web界面参数配置**
- **需求描述**: 通过Web界面动态调整参数，实时预览效果
- **技术实现**: 动态参数Schema，实时验证，拖拽式配置
- **优先级**: P0
- **验收标准**:
  - 参数配置界面直观易用
  - 参数实时验证和错误提示
  - 配置结果可实时预览

**F-607: 多方案对比分析**
- **需求描述**: 并行推演多个方案，对比分析结果
- **技术实现**: 并行执行引擎，对比分析算法，可视化报告
- **优先级**: P1
- **验收标准**:
  - 支持至少3个方案并行推演
  - 生成对比分析报告
  - 识别最优方案

**F-608: What-if分析**
- **需求描述**: 分析参数变化对结果的影响，提供敏感性分析
- **技术实现**: 参数敏感性分析，趋势预测，优化建议
- **优先级**: P1
- **验收标准**:
  - 识别关键影响参数
  - 提供参数优化建议
  - 预测参数变化趋势

**F-609: 实时可视化推演**
- **需求描述**: WebSocket实时推送推演过程，支持交互式可视化
- **技术实现**: WebSocket服务器，实时数据流，交互式图表
- **优先级**: P1
- **验收标准**:
  - 推演过程实时可视化
  - 支持暂停/恢复/重置操作
  - 多客户端并发支持

**F-610: 推演结果持久化**
- **需求描述**: 推演结果自动保存，支持历史结果查询和复用
- **技术实现**: 结果存储系统，查询接口，导出功能
- **优先级**: P2
- **验收标准**:
  - 结果可存储和查询
  - 支持结果导出（JSON/CSV/PDF）
  - 结果复用和比较

#### 19.2 非功能需求

**NF-SIM-001: 性能要求**
- **并发推演**: 支持至少10个方案并行推演
- **响应时间**: 参数配置实时响应<1秒
- **推演时长**: 单次推演平均时长<30秒

**NF-SIM-002: 可靠性要求**
- **推演成功率**: >99%
- **数据一致性**: 版本回退后数据完全恢复
- **沙箱稳定性**: 沙箱环境无内存泄漏

**NF-SIM-003: 安全性要求**
- **数据隔离**: 推演数据与生产环境完全隔离
- **访问控制**: 基于角色的方案访问权限
- **审计追踪**: 所有操作记录完整审计日志

#### 19.3 系统集成需求

**SI-SIM-001: 与Graphiti集成**
- 推演使用临时Graphiti实例
- 复用生产环境的实体本体定义
- 推演结果可选择性回写到生产环境

**SI-SIM-002: 与OPA集成**
- 推演过程中执行OPA策略校验
- 新策略可在推演环境中测试
- 推演方案的合规性验证

**SI-SIM-003: 与决策推荐集成**
- 在模拟环境中推演决策方案
- 对比不同决策策略的效果
- 优化决策参数
