## 1. 项目概述

### 1.1 项目定位

战场情报分析与打击决策系统，基于 Palantir AIP 架构构建的智能决策支持平台。

**核心定位**：
- Graphiti 作为"记忆与认知中枢"（知识图谱层）
- 构建动态、时态化的 Ontology 层
- 实现 OODA 闭环的智能决策系统
- **模拟战场数据生成与训练支撑**

### 1.2 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **基础设施** | Kubernetes, Docker | 自动化部署与资源管理（类似 Apollo） |
| **数据流** | Kafka, Flink | 实时数据管道 |
| **核心图谱** | Neo4j + Graphiti | 知识存储与动态记忆（核心） |
| **数据建模** | Python (Pydantic) | 定义本体与数据清洗 |
| **AI 核心** | OpenHarness + LLM | Agent 编排与逻辑推理 |
| **前端可视化** | React + AntV G6 + CesiumJS | 3D 态势感知与图谱交互 |
| **安全治理** | OPA (Open Policy Agent) | 细粒度权限控制 |
| **多模态处理** | LangChain + OCR + Vision Model | 文档/图像智能解析 |

### 1.3 用户角色

| 角色 | 描述 | 主要功能 |
|------|------|----------|
| **指挥官 (Commander)** | 战场决策者 | 战场态势查看、询问战场信息、下达指令 |
| **情报分析员 (Intelligence)** | 信息处理者 | 情报收集、分析、多模态文档处理 |
| **作战人员 (Operations)** | 执行者 | 执行指令、状态更新 |
| **管理员 (Administrator)** | 系统管理者 | 配置管理、本体管理、用户管理、审计查看 |

|------|------------|----------|
| **Observe** | Intelligence | 感知外部数据（传感器、情报） |

## 3. 功能需求

### 3.1 核心能力

#### F-001: 多智能体协同编排

**描述**: 基于 OODA 环的三 Agent 协同架构

| Agent | 定位 | LLM 模型 | 职责 |
|-------|------|----------|------|
| Commander | 决策中枢 | 强推理模型 | 最终拍板，OPA 校验 |
| Intelligence | 感知+理解 | 快分析模型 | Observe + Orient |
| Operations | 执行中心 | 规划模型 | Act + 回写 |

#### F-002: 双时态知识图谱

**描述**: 基于 Graphiti 的本体知识管理

- **时序推理**：区分"发生时间"与"摄入时间"
- **RAG 增强检索**：混合搜索（语义+关键词+图遍历）
- **事实追溯**：完整的历史时间线
- **增量更新**：秒级数据同步，无需全量重算

#### F-003: OPA 策略治理

**描述**: 统一的策略执行引擎

- **权限校验**：fail-close 原则（不可用时拒绝）
- **Bundle API**：策略热更新，无需重启
- **Rego 策略**：基于属性的访问控制（ABAC）
- **Markdown 策略编辑器**：降低编写门槛

### 3.2 技能层架构

#### F-004: 技能分类体系

| 类别 | 核心技能 | 说明 |
|------|----------|------|
| **感知类** | GraphitiSearch, QueryStatus | 与 Graphiti 交互 |
| **记忆类** | RecordEvent, UpdateStatus | 事件记录 |
| **治理类** | CheckPermission, ValidateAction | 与 OPA 交互 |
| **行动类** | ExecuteStrike, MoveUnit, SendMessage | 外部系统交互 |
| **推理类** | CalculateDistance, PredictOutcome | 复杂计算 |
| **可视化类** | GenerateMapOverlay, SummarizeMission | 报告生成 |

#### F-005: 技能元数据定义

每个 Skill 必须包含：
- **name**: 技能唯一标识
- **description**: LLM 理解的使用说明
- **parameters**: Pydantic 模型定义的输入参数
- **constraints**: 当前 OPA 约束（动态注入）

## 14. 技术研究要点（Alpha 成果）

> 以下是从 req-alpha.md 整合的技术研究成果，作为技术选型的依据。

### 14.1 Graphiti 核心能力

| 能力 | 说明 | 应用场景 |
|------|------|----------|
| **双时态模型** | 区分"发生时间"与"摄入时间" | 回答"系统在 T1 时刻认为 X 在哪里" |
| **增量更新** | 无需全量重算，秒级同步 | 实时战场态势更新 |
| **混合检索** | 语义+关键词+图遍历 | 精准上下文获取 |
| **Episode 机制** | "动作即事件"的建模思维 | 状态变迁的完整历史 |

### 14.2 OPA 与 Graphiti 联动模式

| 模式 | 说明 | 复杂度 |
|------|------|--------|
| **属性透传** | Graphiti 属性直接传给 OPA | 低 |
| **关系注入** | 图遍历结果转化为布尔值 | 中 |
| **OPA 驱动查询** | OPA 返回查询构造器 | 高 |

### 14.3 Router Agent 设计

| 组件 | 职责 | 实现方式 |
|------|------|----------|
| 语义路由 | 初步筛选候选 Skill | 向量相似度 |
| LLM 分发器 | 处理模糊指令，输出结构化 JSON | Pydantic Structured Output |
| OPA 约束注入 | 动态修改 Skill 描述 | 元策略查询 |

### 14.4 Self-Correction 机制

```
执行 -> 捕获错误 -> 反馈给 LLM -> 重试
```

| 组件 | 作用 |
|------|------|
| SkillValidationError | 清晰的错误信息 |
| 历史消息回溯 | 让 LLM 看到错误 |
| 反思-修正循环 | 自我纠错能力 |

### 14.5 OPA 热更新机制

| 机制 | 说明 |
|------|------|
| Bundle API | 定期轮询远程服务器 |
| 策略打包 | tar.gz 格式 |
| 原子加载 | 旧请求用旧策略，新请求用新策略 |
