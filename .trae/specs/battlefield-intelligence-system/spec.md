# 战场情报分析与打击决策系统 - 产品需求文档

## Overview
- **Summary**: 基于graphiti+OPA+skill的战场情报分析与打击决策系统，支持实时战场情报收集、分析、可视化和打击决策。
- **Purpose**: 提供一个智能的战场指挥系统，帮助指挥官实时了解战场态势，做出合理的打击决策，并支持多角色权限管理。
- **Target Users**: 战场指挥官、情报分析师、飞行员等军事人员。

## Goals
- 基于本体论方法，规划系统功能
- 实现战场情报的自动收集和动态更新
- 构建基于图谱的战场实体关系模型
- 提供智能体对话界面，支持战场情况问询和追问
- 实现基于OPA的权限控制和策略拦截
- 支持打击决策的智能推荐和执行
- 实现战场涉及的各类实体的规划和模拟数据构建
- 支持基于图谱预留任务，不阻塞后续逻辑编写
- 交战方大于3个，有清晰的角色定义，战场随机事件模拟，为处置策略提供动态数据
- 提供可视化的界面，查看处置动态
- 构建独立的本体管理功能，本体定义可以以标准格式导入和导出，为本体提供可视化查询界面和本体属性聚合页面，本体版本管理
- 策略执行可模拟，支持版本回退

## Non-Goals (Out of Scope)
- 实际军事行动的执行
- 真实传感器数据的接入
- 复杂的战场模拟物理引擎
- 实时视频流分析

## Background & Context
- 系统基于Python实现，利用graphiti构建战场本体图谱
- 使用OPA进行权限控制和策略管理
- 采用skill机制实现功能模块化
- 支持通过openHarness实现智能体对话界面

## Functional Requirements
- **FR-1**: 基于本体论的系统功能规划
- **FR-2**: 战场情报自动收集与更新
- **FR-3**: 基于图谱的战场实体关系管理
- **FR-4**: 智能体对话界面，支持追问
- **FR-5**: 基于OPA的权限控制和策略拦截
- **FR-6**: 打击决策智能推荐和执行
- **FR-7**: 战场实体模拟数据构建
- **FR-8**: 基于图谱的任务预留机制
- **FR-9**: 多交战方模拟和随机事件生成
- **FR-10**: 可视化界面实现
- **FR-11**: 本体管理功能
- **FR-12**: 策略执行模拟和版本回退

## Non-Functional Requirements
- **NFR-1**: 系统响应时间不超过2秒
- **NFR-2**: 支持多用户并发访问
- **NFR-3**: 系统稳定性和可靠性
- **NFR-4**: 代码可维护性和可扩展性

## Constraints
- **Technical**: Python 3.8+, graphiti, OPA, openHarness
- **Business**: 模拟环境，不涉及真实军事行动
- **Dependencies**: 需要安装graphiti、OPA、openHarness等依赖

## Assumptions
- 战场数据为模拟数据
- 用户角色分为pilot和commander
- 系统运行在本地开发环境

## Acceptance Criteria

### AC-1: 战场情报自动收集与更新
- **Given**: 系统运行中
- **When**: 战场状态发生变化
- **Then**: 系统自动更新战场情报并反映在图谱中
- **Verification**: `programmatic`

### AC-2: 智能体对话界面
- **Given**: 系统启动
- **When**: 用户通过对话界面询问战场情况
- **Then**: 系统返回准确的战场情报，并支持追问
- **Verification**: `human-judgment`

### AC-3: 权限控制和策略拦截
- **Given**: 不同角色用户登录系统
- **When**: 用户尝试执行操作
- **Then**: 系统根据角色权限和策略决定是否允许操作
- **Verification**: `programmatic`

### AC-4: 打击决策执行
- **Given**: 指挥官角色登录系统
- **When**: 指挥官下达打击命令
- **Then**: 系统验证权限和策略后执行打击决策
- **Verification**: `programmatic`

### AC-5: 战场实体模拟数据
- **Given**: 系统启动
- **When**: 系统初始化
- **Then**: 系统加载战场实体模拟数据
- **Verification**: `programmatic`
